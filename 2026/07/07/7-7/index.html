全栈开发工作标准

分为四个方面：需求、开发、测试、审查

1. 需求（Requirements）

  • 需求要规范化成文档，一个功能一个文档
  • 按功能路径组织，例如 student/select scores
  • 文档存放在 docs/requirements/ 目录下
  • 人问 AI 哪个地方需要完善或添加需求，AI 逐个问题问我得到批准，而不是我提出一个 AI 核对一个

2. 开发（Development）

  • 遵循数据库 → 后端 API → 前端 的默认顺序
  • 保持与现有代码风格一致

3. 测试（Testing）

  • 自动化测试用 Playwright E2E
  • 需要肉眼观察界面时，用 noVNC 远程查看浏览器画面

4. 审查（Review）

  • 代码变更后对照需求文档逐项审查
  • 确保每一项需求都已实现
