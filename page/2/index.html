函数依赖＆第一、二、三范式

函数依赖：
部分依赖：复合主键(A,B)，只要一个主键A就能决定另一个列C 就称C部分依赖A
完全依赖：复合主键(A,B)，两个主键都必须存在才能决定另一个列C 就称C完全依赖AB
传递依赖：A -> B -> C，A决定B，B决定C，则称C传递依赖AB
1NF:不能一个表头有多个数值
2NF：完全、传递依赖
3NF：完全依赖

python中cors跨域

cors跨域是什么：解决浏览器同源策略限制，当浏览器发起请求给服务器，服务器响应回来的数据浏览器不接受，因为浏览器认为这是一个跨域请求，所以浏览器会拒绝接受。

针对非简单请求，浏览器会先发起一个预检请求
后端python如何配置：
import flask_cors from CORS
CORS(app)
就自动带上跨域的头信息Access-Control-Allow-Origin:*
辨析：py连db，用的是mysql-connector库，创建游标cursor，执行sql语句，返回结果，关闭连接。
curl的地位是重要发起请求工具
而前端:5000,回到后端：3000是跨域，由后端配置cors

问题2.1找一个老师的所有学生
select
count()
from表一
join表二on 表一.主键=表二.外键
join表三on 表一.主键=表三.外键
where表一.主键=”老师姓名”
group by 表一.主键

问题2.2算出一个老师所有学生的平均分

avg()

group by 表一.主键

昨日理解关系表的概念之一
主键
是一个表的的唯一标识符
外键
一个表的主键在另一个表则称为外键
eg.students.id & scores.id均为两个表的主键；
scores.student_id指向students.student_id为外键
索引
加快查找速度
辨析：没有索引，逐页翻字典；有索引，已创建索引目录，查找时直接定位目标，再摊开数据
eg.PRIMARY KEY (id) –再创建scores表的语句
采用select * from scores where student_id =”S1001”创建索引目录，即S1001=【1,11】，S1001=【2】，S1003=【3】，直接定位到第三行，id=3查找完毕
