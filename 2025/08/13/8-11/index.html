sudo systemctl status mysql && sudo systemctl start mysql

sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

sudo systemctl restart mysql
3检查用户权限
mysql -u root -p
GRANT ALL PRIVILEGES ON . TO ‘your_user‘@’%’ IDENTIFIED BY ‘your_password’ WITH GRANT OPTION;
FLUSH PRIVILEGES;
4防火墙
sudo ufw allow 3306
sudo ufw reload
5 命令行测试
mysql -h your_ip -u your_user -p

函数依赖＆第一、二、三范式

函数依赖：
部分依赖：复合主键(A,B)，只要一个主键A就能决定另一个列C 就称C部分依赖A
完全依赖：复合主键(A,B)，两个主键都必须存在才能决定另一个列C 就称C完全依赖AB
传递依赖：A -> B -> C，A决定B，B决定C，则称C传递依赖AB
1NF:不能一个表头有多个数值
2NF：完全、传递依赖
3NF：完全依赖

python中cors跨域

cors跨域是什么：解决浏览器同源策略限制，当浏览器发起请求给服务器，服务器响应回来的数据浏览器不接受，因为浏览器认为这是一个跨域请求，所以浏览器会拒绝接受。

针对非简单请求，浏览器会先发起一个预检请求
后端python如何配置：
import flask_cors from CORS
CORS(app)
就自动带上跨域的头信息Access-Control-Allow-Origin:*
辨析：py连db，用的是mysql-connector库，创建游标cursor，执行sql语句，返回结果，关闭连接。
curl的地位是重要发起请求工具
而前端:5000,回到后端：3000是跨域，由后端配置cors
